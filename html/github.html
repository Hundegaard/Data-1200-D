Challenges using GitHub

GitHub has been an amazing tool to be able to work and collaborate together as a group and not always having to meet physically or having to transfer our codes to each other using a memory stick or some other way.

As practical and useful as GitHub has been, we've run into a handful of problems we could've avoided had we been more experienced with GitHub and how the branching system works. First, we had some issues setting up the GitHub collab and making sure that everyone had all the files and folders we had set up. Next came all the merger conflicts due to not pulling and pushing our updates often enough, more than one person working on the same thing, or it just emerged because we were all working in the same file.

All of this leading to various problems such as:

  • Complete paragraphs of code went missing, so we had to go back in time to a past commit to try and scour and salvage whatever went missing and then having to implement it in the new commit again.

  • Duplication of code elements, which probably happened because two people were working inside the same section of an element. Leading to complete chaos when both of those were merged.

  • Someone being behind 5 updates and not getting any pull requests, we have still no idea how this happened or how we resolved this. We just tried a lot of different things praying and hoping it would be fixed, and it somehow did.

  • Pushing an old version so everyone who pulled it went back, this being related to the issue above. Someone being unaware they're behind and then pushing their old version to the rest of the group, so we had to revert a lot of times due to this.
