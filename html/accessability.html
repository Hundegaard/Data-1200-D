Accessability

Accessibility Review of Our Website Design

Accessibility is essential to ensure all users, regardless of ability, can interact with and benefit from a website. In designing our site, we focused on universal design principles to create an inclusive, user-friendly experience. Below is a summary of the accessibility features we implemented, along with insights from our automated and manual testing processes.

Key Accessibility Features

  • Readable Fonts

    We selected a clear, legible font to make content accessible for users with visual impairments or cognitive disabilities. This ensures readability across devices and screen resolutions.

  • Relative Contrast in Colors

    The colors on our site provide strong contrast between text and background. For example, the light pink color avoids blending into white backgrounds, and it darkens when hovered over, improving visibility for interactive elements.

  • Text Scaling

    The site supports text scaling on different screen sizes without breaking the layout, ensuring readability for users who rely on zooming or magnification tools.

  • Spacious Layout

    Generous spacing between text blocks and interactive elements enhances usability for users with motor impairments and cognitive challenges.

  • Minimal Use of Divs

    We reduced unnecessary div elements in the HTML, making it easier for assistive technologies like screen readers to interpret the content.

Automated Testing

Using Google's Lighthouse tool, we assessed our site's accessibility, it gave us scores based on on different aspects of the website that helped give feedback for possible improvements. The report highlighted several issues, and we addressed the following:

  • Logo Display

    We fixed an issue where the logo did not display correctly on some devices by ensuring it had the proper width and height properties.

  • Spacing Issues

    We resolved spacing issues that were impacting the layout and navigation, improving the visual structure of the site.

  • Explicit Width and Height for Images

    We ensured that all images, including the logo, have explicit width and height properties, preventing layout shifts and improving page stability.

Manual Code Review and Improvements

Beyond automated testing, we performed a manual review to ensure accessibility:

  • Hover Color Contrast

    We manually confirmed that the hover state contrast was sufficient for visually impaired users.

  • Semantic HTML

    We verified the proper use of semantic HTML elements like header, main, and footer to ensure assistive technologies can interpret the page effectively.

  • Screen Reader Testing

    We tested navigation with screen readers to ensure layout shifts did not disrupt usability and all labels were read correctly. The webpages were tested on Mobile M (375px), Laptop (1024px), and 4K (2560px)

In conclusion, our site's accessibility score reflects thoughtful design decisions and improvements. Our manual testing helped refine features like font readability, hover states, and semantic HTML to ensure a more inclusive user experience.
